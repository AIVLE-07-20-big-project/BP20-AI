# 온라인 검색어 기반 수요 예측 (Online Trend Search)

고객이 온라인에서 검색하는 키워드의 검색량 추이를 분석해, 특정 메뉴·상품의 수요 증가 가능성을 사전에 포착하고 재고 조정을 추천하는 기능입니다.

## 실행 결과 예시

> "최근 '초코케이크' 검색량이 증가하고 있습니다. 재고를 평소보다 10% 늘리는 것을 추천합니다."

업종을 선택하면, 대표 메뉴의 검색 트렌드를 분석해 급증/증가/평이/감소 신호와 함께 자연어 요약, 표, 장기 추이 그래프가 담긴 HTML 리포트를 생성합니다.

---

## 사전 준비

### 1. 라이브러리 설치

```bash
pip install -r requirements.txt
```

### 2. 네이버 API HUB 인증키 발급

1. [NAVER Cloud Platform](https://www.ncloud.com)에서 계정 생성
2. [NAVER API HUB](https://apihub.naver.com) 포털에서 "Search Trend" 상품 신청
3. 애플리케이션 등록 후 Client ID / Client Secret 발급

> 참고: 기존 "네이버 개발자센터"(developers.naver.com) 방식은 2027-06-30부로 서비스가 완전히 종료되므로, 반드시 NAVER API HUB(NCP) 방식으로 신청해야 합니다.

### 3. 환경변수 설정

`.env.example`을 복사해 `.env` 파일을 만들고 값을 채워주세요.

```bash
cp .env.example .env
```

---

## 실행 방법

```bash
python search_trend_by_industry.py
```

실행하면 지원 업종 목록이 나오고, 업종을 입력하면 됩니다.

```
=== 지원하는 업종 목록 ===
  - 카페
  - 한식당
  - 분식집
  - 치킨집
  - 중식당
  - 일식당
  - 피자·양식점

업종을 입력하세요: 카페
```

실행이 끝나면 `daily_report_{업종}_{날짜}.html` 파일이 생성됩니다. 브라우저로 열어서 확인하세요.

## 전체 작동 흐름

```
① 업종 선택
        ↓
② 후보 메뉴군 중 검색량 상위 10개 자동 선정 (7일 캐시)
        ↓
③ 선정된 메뉴의 최근 3개월 검색 트렌드 조회
        ↓
④ 저검색량 메뉴 자동 필터링
        ↓
⑤ 증가율 계산 및 신호 판정 (급증/증가/평이/감소)
        ↓
⑥ 자연어 요약 문장 생성
        ↓
⑦ 신호 있는 메뉴의 10년 장기 추이 그래프 생성
        ↓
⑧ HTML 리포트 생성 및 저장
```

## 사용 API

| API | 용도 |
|---|---|
| NAVER API HUB - Search Trend | 메뉴 선정, 검색 추이 조회, 장기 추이 조회 |

엔드포인트: `https://naveropenapi.apigw.ntruss.com/datalab/v1/search`

## 검증 결과

"검색량 증가가 실제 매출 증가와 연관된다"는 핵심 가설을 두 가지로 검증:

1. **가상 데이터 검증**: 계절 상품 평균 상관계수 0.59~0.80 / 대조군 0.05
2. **실제 공공데이터 검증**: 서울시 카페 업종 매출 vs "카페" 검색지수 (2021~2026 분기) → **상관계수 0.852**

## 파일 구성

| 파일 | 역할 |
|---|---|
| `search_trend_by_industry.py` | 메인 실행 파일 |
| `requirements.txt` | 의존 라이브러리 목록 |
| `.env.example` | 환경변수 형식 (실제 키는 미포함) |
| `menu_selection_cache.json` | 실행 시 자동 생성되는 캐시 (git에는 포함 안 함) |
| `daily_report_*.html` | 실행 시 자동 생성되는 산출물 (git에는 포함 안 함) |

## 알려진 한계

- 후보 메뉴 목록 자체는 사전 정의된 값 (완전 자동 발견은 아님)
- 저검색량 메뉴(니치한 이름)는 분석 대상에서 자동 제외됨
- 개별 매장 단위 검증은 아직 진행되지 않음 (업종 단위 실데이터 + 가상 매장 데이터로만 검증)
